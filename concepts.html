Concepts

In JavaScript when you want to do some asynchronous things, you usually use the window.setTimeout function. However it is rather limited. It only allows you to start text in a string or code inside a function to run after a defined set of milliseconds. Some browsers offer the posibility to give arguments to a function, others don't. Also, window.setTimeout cannot be used within objects, because the value of this is lost. For this purpose I created ASyncQuery.js.

Window.setTimeout alternative,

ASyncQuery.js was written as a window.setTimeout alternative. 
function A_Simple_Demo_1(){
	var an_Object = new Object();
	an_Object.message = "I am a property of an object. ";
	ASyncQuery(function(){alert(this.message);}, 1000, {Scope: an_Object});
}
Test example #1. It allows to use this inside a function, as you can se in this example. Also ASyncQuery can give a function a number of arguments, by specefiing the arguments parameter.

And more ...

There's even more in it already. You can:
  • let items wait for others to finish,
  • let things happen in case of errors,
  • delete individual items,
  • raise events and let them start functions
Thers's even more planned. Soon you'll be able to:
  • let queries run under custom conditions only and let them resume automatically once a condition is done.
  • load external JS files and wait for them to load
