API Documentation

General

This is a API reference for ASyncQuery.js. You will find all possible functions and settings here.
Functions written italic refer to an instance of the specific type.

Including the file

You may include the javascript library file like any other in your web page. Simply insert: 
<script src="js/ASyncQuery.js" type="text/javascript" language="JavaScript"></script>
inside the <head> or <body> tag of your website and put js/ASyncQuery.js in the appropriate folder. You can also rename the file or put it in another folder, but remember, you will have to adapt the <script> tag in that case.

Debug Mode

ASyncQuery.js has a debug mode. To enable it, put an html comment containg the following in the head of your html document before the script tag including the library:
<!-- ~ASyncQuery.js: Debug=True~ -->
The Debug Mode allows you to use the (undocumented) ASyncQuery.Debug() function.

API Options

You might find something like:
<object> setting1, ..., <object> options
in the API documentation. This references to settings. This means that you have to look at the end of the section for different options which are possible. In the table at the end of the section all settings are listed, ranked by importance and types. When calling the function, setting1 will refer to the first option listed in the table with the it's type. setting2 will refer to the second one of that type unless it is of another type. The options variable has to be a hash containing the settings by Name. The hash overrides the settings before.

Data Types

There are 2 new DataTypes introduced in ASyncQuery.js:
  • ASyncQuery: A type containing a query (list) of different ASyncQueryItems which are all worked on one after another. obj instanceof ASyncQuery.Type.ASyncQuery should return true for all objects of this type.
  • ASyncQuery.QueryItem: A type containing a function and all it's parameters. obj instanceof ASyncQuery.Type.ASyncQueryItem should return true for all objects of this type.

ASyncQuery(<function> Func, [<number> Timeout = 0], [<object> setting2, ...], [<object> ASyncQueryItemOptions])
Asynchronously starts a function. Similar to window.setTimeout(), but with more advanced functions.
Returns ASyncQuery.QueryItem.
<function> Func Function to call. Required.
<number> Timeout Optional variable containing a number of milliseconds to wait before starting the function. Default is 0.
(further parameters see ASyncQuery.QueryItem)
<object> QueryItemOptions Optional map containing different QueryItemOptions.

ASyncQuery Options

Type Name Default Effect
<string> name "" Name of the Query. May contain any supported character.
<number> defaultPriority 0 Default priority for Query Items.
<number> defaultPersistence 0 Default persistence for Query Items.
<number> defaultTimeout 0 Default timeout for Query Items.
<boolean> autoStart true Determines if the query will be started automatically if a new QueryItem is attached.
<function> onError function(){} A function to call if an unhandled error occurs. A parameter containing the Error will be passed to the function.
<function> onIdle function(){} A function to call every time when the ASyncQuery becomes idle, meaning the last Item is parsed.
<object> defaultScope window Default Scope for Query Items. Can only be specefied in option map.

ASyncQueryItem Options

Type Name Default Effect
<string> name randomly generated UUID-like string.
Contains 8 10-character sequences, which are connected by "-"s. Each sequence can contain numbers, upper and lower case letters from a to z.
Example:
mP78m86otb-0FGEKzSAoF-hL2M5IaRAa-u8ksyG5VKX- IuIcXVbJEe-ZNSXw69xMk-Z2KMuI4nJX-JTv4VOnRGL
Name of the Item. May contain any supported character.
<string> handlesEvent "" If not "", the Item is registered as an event handler for the specefied Event.
<number> persistence as specefied in ASyncQuery defaultPersistence option, otherwise 0 Determines how many times the Item will be repeated. 0 means no repeat at all. Ignored for Event Handlers unless changed later with .option()
<number> priority as specefied in ASyncQuery defaultPriority option, otherwise 0 Priority of the Item. The higher the priority, the earlier it will be ran in it's Query.
<number> timeout as specefied in ASyncQuery defaultTimeout option, otherwise 0 Minimum time to wait until the item will be started. Only important for items which are in a query.
<array> arguments [] An array of parameters to pass to the Item when it is called.
<array> QueryItems [] An array of names of other Query items which have to have been started before his one will start.
<function> function function(){} Function of the item. It will be ran when the item is called.
<function> onError function(e){throw(e);} A function to call if an unhandled error occurs. A parameter containing the Error will be passed to the function. It overrides the onError Option of the Items Query.
<function> onDeath function(){} A function to call after the item is no longer called.
<object> Scope as specefied in ASyncQuery defaultScope option, otherwise window Value of this inside the function. Can only be specefied in the ASyncQueryItemOptions parameter.
